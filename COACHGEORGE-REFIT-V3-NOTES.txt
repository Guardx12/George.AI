CoachGeorge refit v3

What changed:
- Rebuilt /coachgeorge around the new product direction: Coach George is a daily transformation coach beside NutriCheck/MyFitnessPal, not an exact calorie-tracking or meal-plan app.
- Removed the old forced age/sex/height setup experience from the main flow.
- Added premium app-style home screen with a large voice orb, quick coaching actions, dashboard cards, and app-like layout.
- Added editable Coach Profile panel: goal, current weight, target weight, training focus, biggest struggle, food style, tracker app, optional calorie/protein targets, and coaching style.
- Added Active Phase model with archived phases so users can change direction without deleting history.
- Added safe Change/Reset controls: start new phase, reset current phase only, or delete all data with double-click confirmation.
- Added local structured memory: profile, active phase, archived phases, progress logs, coaching notes, and useful summaries.
- Rewrote George's coaching replies around the new jobs: meal guidance, hunger, bad days, scale panic, training, tracker interpretation, motivation, safety, and reset guidance.
- Kept voice behaviour browser-safe: user taps first, then George speaks.
- Added typed chat input so the app still works even if voice is blocked or unavailable.

Important:
- This is still localStorage-based for MVP/demo use. For production, move profile, phase, memory, progress, and subscription state into Supabase or another backend.
- Full transcripts are not the main memory. George stores useful summaries/notes instead.
- The old meal-plan/nutrition engine files are still in the repo, but the CoachGeorge UI no longer depends on exact meal-plan generation.
- I could not run a full Next build in this environment because node_modules/Next were not installed in the extracted ZIP, but I ran a TypeScript transpilation syntax check against the rewritten TSX component.
